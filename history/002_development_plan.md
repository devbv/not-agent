# 코딩 에이전트 개발 계획

## Phase 1: 기초 인프라 구축
### 1.1 프로젝트 셋업
- 개발 언어/프레임워크 선택 (Python, TypeScript 등)
- 프로젝트 구조 설계
- 개발 환경 설정

### 1.2 LLM 통합
- Claude API 또는 다른 LLM API 연동
- 기본 프롬프트 → 응답 파이프라인 구축
- API 키 관리 및 인증 처리

### 1.3 기본 CLI 인터페이스
- 사용자 입력 받기
- 응답 출력하기
- 대화 세션 관리

---

## Phase 2: 핵심 도구(Tools) 구현
### 2.1 파일 시스템 도구
- `Read`: 파일 읽기
- `Write`: 파일 쓰기
- `Edit`: 파일 수정 (diff 기반)
- `Glob`: 패턴으로 파일 찾기
- `Grep`: 파일 내용 검색

### 2.2 실행 도구
- `Bash`: 쉘 명령어 실행
- 타임아웃 처리
- 출력 캡처 및 포맷팅

### 2.3 도구 실행 엔진
- Tool Use 프로토콜 구현
- 도구 호출 파싱 및 실행
- 결과를 LLM에 피드백

---

## Phase 3: 에이전트 루프 구현
### 3.1 기본 에이전트 루프
```
사용자 입력 → LLM 호출 → 도구 실행 → 결과 피드백 → 반복
```

### 3.2 대화 컨텍스트 관리
- 메시지 히스토리 유지
- 컨텍스트 윈도우 관리
- 요약/압축 기능 (긴 대화용)

### 3.3 사용자 질문 기능
- 에이전트가 사용자에게 질문하기
- 선택지 제공
- 확인 요청 (위험한 작업 전)

---

## Phase 4: 코드 생성 및 테스트
### 4.1 코드 생성
- 언어별 코드 생성 최적화
- 기존 코드 스타일 파악 및 적용
- Import/의존성 관리

### 4.2 자동 테스트
- 테스트 코드 생성
- 테스트 실행 및 결과 파싱
- 실패 시 자동 수정 시도

### 4.3 코드 검증
- 문법 오류 검사
- 타입 체크 (해당 언어)
- 린팅

---

## Phase 5: 고급 기능
### 5.1 멀티 에이전트
- 서브 에이전트 생성 및 관리
- 병렬 작업 처리
- 에이전트 간 통신

### 5.2 계획 모드
- 복잡한 작업 분해
- 단계별 계획 수립
- 사용자 승인 후 실행

### 5.3 웹 기능
- 웹 검색

## Phase 6: 사용자 경험 개선
### 6.1 진행 상황 표시
- 실시간 스트리밍 출력

### 6.2 안전장치
- 위험 명령어 확인
- 샌드박스 실행

## 기술 스택 제안

### Option A: Python
- **장점**: AI/ML 생태계 풍부, 빠른 프로토타이핑
- **프레임워크**: LangChain, LlamaIndex, 또는 직접 구현
- **CLI**: Click, Rich (터미널 UI)

### Option B: TypeScript/Node.js
- **장점**: Claude Code와 동일 스택, npm 생태계
- **프레임워크**: Vercel AI SDK, 또는 직접 구현
- **CLI**: Commander, Ink (React for CLI)

### Option C: Rust
- **장점**: 성능, 단일 바이너리 배포
- **단점**: 개발 속도 느림
- **적합**: 프로덕션 레벨 도구
